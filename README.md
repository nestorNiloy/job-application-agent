# AI-Driven Autonomous Job Application & Tracking Agent

An asynchronous FastAPI backend that automates the tedious parts of job
hunting: discovering **real, live listings via the Adzuna Job Search API**,
tailoring application notes with a rule-based skill-matching engine, and
tracking every application through a status state machine with
human-in-the-loop authorization before anything is marked as applied. A
minimal Kanban-style frontend visualizes the pipeline in real time.

## Architecture

```
Adzuna Job Search API (real, live listings)
            │
            ▼
   ┌─────────────────────────────┐
   │   FastAPI Backend           │
   │                             │
   │  Router Layer  (main.py)    │
   │        ▼                    │
   │  Service Layer (services.py)│  ← scraping + AI tailoring
   │        ▼                    │
   │  Repository Layer (crud.py) │
   └──────────────┬──────────────┘
                  ▼
        SQLite database (jobs_agent.db)
```

**Layers**
- `app/models.py` — SQLAlchemy 2.0 async models, `JobStatus` enum, engine/session setup
- `app/schemas.py` — Pydantic v2 request/response validation schemas
- `app/crud.py` — Repository layer; all raw DB access lives here
- `app/services.py` — Business logic: real Adzuna scraper + rule-based tailoring engine
- `app/main.py` — FastAPI routes, lifespan startup, global 404 exception handler
- `static/index.html` — Kanban board frontend (vanilla JS, no build step)

## Data Model

`JobApplication`

| Field | Type | Notes |
|---|---|---|
| id | int | Primary key |
| company_name | str | Required |
| position_title | str | Required |
| job_board | str | e.g. "LinkedIn", "Xing" |
| job_description_text | text | Raw scraped text, feeds the AI tailoring step |
| status | enum | SCRAPED → PENDING_REVIEW → APPLIED → INTERVIEWING / REJECTED / OFFERED |
| applied_date | date | Set automatically on transition to APPLIED |
| ai_tailored_notes | text | Rule-based tailoring output (optional) |
| external_id | str | Adzuna's own listing id — used to skip duplicates on re-scrape |
| source_url | str | Link to the live listing on Adzuna |

## API Endpoints

| Method | Path | Body | Description |
|---|---|---|---|
| POST | `/api/jobs/scrape` | `{keywords, location}` | 202 Accepted — queues a real Adzuna search in the background |
| GET | `/api/jobs` | — | All jobs, newest first (feeds the Kanban board) |
| GET | `/api/jobs/pending` | — | Jobs sitting in `PENDING_REVIEW` |
| PUT | `/api/jobs/{id}/apply` | — | Transitions a job to `APPLIED`, stamps `applied_date` |
| PUT | `/api/jobs/{id}/status` | `{new_status}` | Manual status update (INTERVIEWING, REJECTED, etc.) |
| PUT | `/api/jobs/{id}/tailor` | — | Runs the rule-based tailoring engine, saves the notes |
| GET | `/api/jobs/metrics` | — | Total counts grouped by status |

Interactive docs are auto-generated by FastAPI at `/docs` once running.
The Kanban board frontend is served at `/ui/`.

## Running locally

```bash
pip install -r requirements.txt
cp .env.example .env   # add your Adzuna credentials
uvicorn app.main:app --reload
```

Visit `http://127.0.0.1:8000/docs` for the Swagger UI, or
`http://127.0.0.1:8000/ui/` for the Kanban board.

## Design notes

**Scraping is real, not mocked.** `ScraperService` calls the live Adzuna
Job Search API and returns actual current job listings. Each listing's
Adzuna id is stored as `external_id` so re-running a scrape with the same
keywords never creates duplicate records — only genuinely new listings
are inserted.

**Tailoring is a real algorithm, honestly labeled.** `TailoringService`
is a deterministic keyword-overlap matcher between the job description
and a configurable resume skill list — it is not a live LLM call, and the
output says so explicitly. The method signature is intentionally kept
swappable: replacing the body of `generate_tailored_notes` with a real
OpenAI/Gemini/Claude API call is a drop-in change that doesn't touch the
router or repository layers.

## Tech stack

Python 3.11+, FastAPI, SQLAlchemy 2.0 (async), SQLite via `aiosqlite`,
Pydantic v2, httpx, Uvicorn, vanilla JS frontend.
